# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in AicodeX, please report it by emailing the maintainer or opening a private security advisory on GitHub.

**Please do not report security vulnerabilities through public GitHub issues.**

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if available)

### Response Timeline

- We will acknowledge your report within 48 hours
- We will provide a more detailed response within 7 days
- We will work on a fix and coordinate disclosure

## Security Best Practices

When using AicodeX:

1. Keep dependencies up to date
2. Use the latest stable version
3. Review configuration for sensitive data
4. Follow principle of least privilege
5. Enable security features in production

## Known Security Considerations

- Hotkey handling requires appropriate permissions
- Configuration files may contain sensitive data
- Docker images should be from trusted sources
- Dependencies are regularly scanned for vulnerabilities
